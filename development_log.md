## Challenges Faced

### 1. Tailwind CSS v4 Breaking Changes
**Problem:** `npx tailwindcss init -p` failed. Directives like `@tailwind base` were not recognized.
**Solution:** Switched to `@import "tailwindcss"` and `@tailwindcss/postcss` plugin. Configured dark mode via `@custom-variant dark` instead of `tailwind.config.js`.

### 2. Gemini Model Availability
**Problem:** `gemini-1.5-flash` returned "model not found" errors.
**Solution:** Updated to `gemini-2.0-flash` across all service files.

### 3. API Quota Limits
**Problem:** Free tier quota exceeded caused 429 errors.
**Solution:** Built comprehensive fallback system with 5 local AI agents that generate realistic data when the API is unavailable.

### 4. Chatbot Generic Persona
**Problem:** AI responded as "I am an AI assistant" despite prompts.
**Solution:** Added a `SYSTEM_PROMPT` constant in `geminiService.js` with explicit rules about identity and behavior. Built keyword-based local fallback that enforces the Student Success OS persona.

### 5. Dashboard Missing Sections
**Problem:** Earlier versions of Dashboard only showed 2-3 sections.
**Solution:** Complete rewrite with all 7 required sections using a grid layout system.

### 6. PDF Export Quality
**Problem:** html2canvas captured low-resolution output on dark backgrounds.
**Solution:** Set explicit `scale: 2`, `backgroundColor`, and `useCORS: true` options.

## Lessons Learned

1. **Always specify Tailwind version** â€” v3 and v4 have incompatible APIs. The `@import` vs `@tailwind` directive change is a breaking difference.
2. **Graceful degradation is essential** â€” When depending on external APIs, always build a fallback. The 5-agent fallback system meant the app works even without internet.
3. **System prompts need repetition** â€” Getting an AI to consistently follow a persona requires explicit rules in every prompt, not just the initial instruction.
4. **Component co-location** â€” Keeping Dashboard as one file for 7 sections was manageable up to ~400 lines. For further growth, consider splitting into sub-components.
5. **localStorage is perfect for this use case** â€” 10 reports at ~5KB each is well within the 5MB limit, and avoids backend complexity.
6. **PDF generation from DOM** â€” `html2canvas` + `jsPDF` works well but requires careful scaling for quality. Prefer `scale: 2` for retina-quality output.
